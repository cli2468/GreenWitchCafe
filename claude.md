# Project Constitution (claude.md / gemini.md)\n\n## Data Schemas\nAs this is a static website for a cafe (rather than an automated data pipeline), the "Data Schema" refers to our File Structure and Content Management approach:\n\n- **Input**: Local static assets (Images, Logos) in `brand/` and content data arrays (`src/data/`).\n- **Output**: React Single Page Application (SPA) bundled via Vite, hosted on GitHub Pages or Vercel.\n\n## Behavioral Rules & Design Principles\n1. **Mobile-First**: The majority of users looking for a cafe are on their phones (checking hours, menu, or location). Navigation must be thumb-friendly.\n2. **Accessible (WCAG)**: High contrast text, alt-tags for all images, and clear legible typography.\n3. **Visual-Heavy & Experiential**: Prioritize high-quality photography of the space, fresh juices, and plant-forward food. The website must evoke the "peaceful" and "refreshing" physical atmosphere.\n4. **No-Friction Calls to Action (CTAs)**: "View Menu" and "Visit Location" must be prominent.\n5. **Brand Voice Constraint**: Copy must be nurturing, earthy, and inviting ("Cool Witch" persona). \n   - **Use Words Like**: Refreshed, peaceful, intentional, plant-forward, coven, health-benefit.\n   - **Avoid Words Like**: Fast-food, transactional, substitute, bland.\n\n## Architectural Invariants\n- **Zero External Integrations (v1)**: No databases, CMS, or complex backend logic yet.\n- **Handoff Ready**: The final build must be easily portable from your GitHub to the client's final hosting provider.\n
